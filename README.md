# Git Workspace Diff Launcher 仕様メモ

## 1. 目的

cmux ベースの開発環境で、VS Code / Cursor の左サイドバーにあったような「今どれくらい変更が入っているか」を素早く把握できるようにする。

このアプリの主目的は Git 操作ではなく、**複数リポジトリの変更状況の可視化**である。

特に、1つの案件が複数の Git リポジトリに分かれている場合に、workspace 全体を俯瞰して「どの repo にどれくらい変更があるか」を素早く確認できるようにしたい。

---

## 2. 前提となる使い方

* 普段の開発は cmux 上で行う
* ブランチ作成、commit、push などの Git 操作は基本的に AI にやらせる想定
* 自分が見たいのは主に以下

  * 変更があるかないか
  * どの repo に変更があるか
  * 各 repo にどれくらい変更があるか
  * 必要なら軽く diff を見る
* SourceTree 的なフル機能 Git クライアントは不要
* 監視・可視化・軽い確認に特化したい

---

## 3. ほぼ確定していること

### 3-1. アプリの形

* macOS アプリとして実装する
* 常駐型のメイン UI ではなく、**ランチャー型**にする
* グローバルショートカットキーで呼び出せるようにする
* 呼び出したら小さいパネルが前面に出て、閉じたら消える
* 操作感は Spotlight / Raycast / Alfred のようなイメージに寄せたい

### 3-2. 主な機能

* cmux で開いている workspace を元に対象ディレクトリを決定する
* その workspace 配下を走査し、Git repo を検出する
* 各 repo について変更状況を一覧表示する
* 一覧では最低限以下を表示する

  * repo 名
  * 変更数
* できれば branch 名も表示したい
* 変更数が多い repo や、変更がある repo を優先的に見やすくしたい
* 一覧から repo を選択して詳細を見られるようにしたい
* 詳細では最低限 `git diff --stat` 相当の情報を見られるようにしたい
* 必要なら通常の diff まで見られる設計にしたいが、優先度はそこまで高くない

### 3-3. 対象となる repo の見つけ方

* workspace 配下を再帰的に探索する
* `.git` を持つディレクトリを Git repo とみなす
* `.git` がディレクトリではなくファイルであるケースもあり得るので、できれば対応したい
* 重くなりやすいディレクトリは探索対象から除外したい
  例:

  * `node_modules`
  * `vendor`
  * `dist`
  * `build`
  * `.next`
  * `tmp`
  * `coverage`
  * `Pods`

### 3-4. 変更数の取得方法

* 基本は `git status --porcelain` を使う
* 行数を変更数として扱う
* まずは staged / unstaged / untracked の区別はしなくてよい
* 将来的には区別できるように拡張可能な実装にしたい

### 3-5. UI / UX の基本方針

* 起動は一瞬で終わること
* 余計な機能を持たず軽量であること
* Git クライアントではなく「workspace の dirty 状態ビューア」であることを徹底する
* キーボード操作中心で使えること
* `Esc` ですぐ閉じられること
* repo 名で絞り込み検索できるとよい

### 3-6. 優先度

MVP では以下が最優先。

* グローバルショートカットで呼び出せる
* 現在の workspace を取得できる
* 配下の repo 一覧を出せる
* 各 repo の変更数を表示できる
* 変更あり repo を上に出せる
* repo を選ぶと詳細を見られる

## 4. MVP として実装したい内容

### 4-1. 起動

* グローバルショートカットでランチャーを開く
* ランチャーは前面に表示される
* フォーカスは検索欄または一覧に当たる
* `Esc` で閉じる

### 4-2. 一覧画面

* 現在の workspace 名またはパスを表示
* repo 一覧を表示
* 各行に最低限以下を表示

  * repo 名
  * 変更数
* 可能なら以下も表示

  * branch 名

### 4-3. ソート

* 変更あり repo を上に表示
* その中で変更数が多い順
* 同数なら名前順

### 4-4. 詳細画面

* repo を選択して詳細表示
* 詳細では最低限以下を表示

  * repo 名
  * branch 名
  * 変更数
  * `git diff --stat` 相当の情報

### 4-5. 検索

* repo 名でフィルタできる

## 5. 実装方針として有力なもの

### 5-1. 技術スタック

第一候補:

* Swift
* SwiftUI
* 必要に応じて AppKit 併用

理由:

* macOS ネイティブで軽量に作れる
* ランチャー型 UI と相性が良い
* グローバルショートカットや前面表示などを扱いやすい

### 5-2. Git 情報取得

* 内部で git コマンドを実行して情報を取得する
* 例:

  * `git status --porcelain`
  * `git branch --show-current`
  * `git diff --stat`

### 5-3. 更新方法

* 最初は呼び出し時に都度スキャンしてよい
* 将来的にはキャッシュや差分更新を入れてもよい

## 6. 未確定・これから詰めること

### 6-1. cmux から workspace をどう受け取るか

ここは未確定。

候補:

1. cmux 起動時に current workspace の path をファイルに書き出す
2. tmux / cmux の current pane path などから推定する
3. アプリ側で workspace を手動指定する
4. 最近使った workspace の中から選ぶ

現時点では、**cmux 側からファイルに書き出して、それをアプリが読む方式**が有力。

例:

* `~/.cmux/current-workspace`

### 6-2. 一覧画面と詳細画面のUI構成

未確定。

候補:

* 一覧だけをまず出し、Enter で詳細に遷移
* 一覧と詳細の2ペイン表示
* 一覧選択時に下部プレビュー表示

まずはシンプルに、

* 一覧
* Enter で詳細
* Back / Esc で戻る
  がよさそう

### 6-3. diff の粒度

未確定。

候補:

* `git diff --stat` のみ
* changed files の一覧も表示
* 通常の diff 本文も表示

優先度としては

1. `git diff --stat`
2. changed files 一覧
3. 通常 diff
   の順

### 6-4. 変更数の定義

現時点では `git status --porcelain` の行数でよいが、将来的には以下をどう扱うか詰める余地がある。

* untracked を含めるか
* staged / unstaged を分けるか
* ignored はどうするか
* submodule や worktree をどう扱うか

### 6-5. 複数 workspace 対応

未確定。

MVP では「現在の workspace 1つ」でよいが、将来的には

* 最近使った workspace の切り替え
* 複数 workspace の一覧表示
* 案件ごとの切り替え
  はあり得る

### 6-6. 更新タイミング

未確定。

候補:

* ランチャー表示時のみ更新
* 表示中は数秒ごとに再取得
* ファイル監視ベースで更新

MVP では

* 表示時に取得
* 表示中に必要なら手動リフレッシュ
  でもよい

### 6-7. どこまで操作機能を持つか

現時点では Git 操作機能は不要。

ただし将来的に以下のような補助機能はあり得る。

* repo のパスをコピー
* Finder で開く
* ターミナルで開く
* エディタで開く

ただし commit / push / branch 操作などは本アプリの主目的から外れるので、原則入れない方向でよい

### 6-8. 大規模 workspace への対応

未確定。

考慮点:

* repo 数が多いと重い
* 深い再帰探索が重い
* `git status` が遅い repo がある

将来的な対策候補:

* 探索除外ルールの強化
* repo のキャッシュ
* 並列実行
* タイムアウト
* 前回結果の再利用

---

## 7. このアプリがやらないこと

以下は少なくとも当面は対象外。

* commit
* push
* branch 作成
* merge / rebase
* stash 操作
* PR 操作
* GitHub / GitLab 連携
* SourceTree 的なフル Git クライアント化
* 単一 repo の詳細な Git 履歴閲覧を主目的にすること

本アプリはあくまで
**workspace 配下の複数 repo の変更状況を素早く確認するための軽量ランチャー**
である。

---

## 8. 期待する体験

理想的な利用フローは以下。

1. cmux で作業している
2. AI が複数 repo に変更を入れる
3. 「今どこがどれくらい変わった？」と思う
4. グローバルショートカットでランチャーを開く
5. workspace 配下の repo 一覧と変更数が一瞬で見える
6. 気になる repo を選ぶ
7. 必要なら diff stat を確認する
8. `Esc` で閉じて cmux に戻る

---

## 9. 注意

* Git クライアントを作ろうとしないこと
* 監視・可視化・軽い確認に特化すること
* UI は軽く、起動は速く、操作は少なくすること
* macOS のランチャーアプリとして自然な体験を優先すること
* MVP では機能を盛りすぎないこと
* まずは「workspace 配下の repo 一覧 + 変更数 + 軽い詳細」までを確実に作ること
